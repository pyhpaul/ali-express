# AliExpress 评分增强设计

日期：2026-05-08

## 目标

增强现有 DrissionPage MVP 的 `rating` 采集能力，使搜索列表页中已经展示的评分数字能够进入 `products.csv`，并保留可选详情页补全能力作为低频兜底。

## 现状证据

真实页面 `https://www.aliexpress.com/w/wholesale-women-dress.html` 的商品卡片文本包含以下结构：

```text
商品标题
$0.99
$9.92
-90%
4.2
329 sold
```

当前脚本只把包含 `star`、`rating`、`reviews`、`评价` 等词的行当作评分，因此独立数字行 `4.2` 被漏掉。

## 方案

第一步只增强列表页：

- 在浏览器端脚本中从商品卡片文本行中识别独立评分数字。
- 评分候选必须是 `1` 到 `5` 或一位小数，例如 `4`、`4.2`、`5`。
- 排除价格、折扣、销量等行；评分优先取出现在销量行之前的最后一个评分候选。

第二步添加可选详情页补全参数：

- CLI 增加 `--enrich-detail-rating` 和 `--detail-limit`。
- 默认关闭，不增加采集请求。
- 打开后只对 `rating == 0` 的前 N 个商品尝试访问详情页提取评分。
- 详情页失败不阻断主流程。

## 边界

- 不做批量并发详情页访问。
- 不承诺每个商品都有评分；没有评分时保留 `0.0`。
- 不改变 CSV 字段结构。

## 验证

- 单元测试覆盖列表文本评分提取。
- 单元测试覆盖 CLI 新参数。
- 真实运行 `women dress`，检查 `products.csv` 中 `rating` 不再全为 `0.0`。
